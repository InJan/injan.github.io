Mar 16, 2017 | Hits

一、研究背景及意义

1.Chinese Students and Scholars Association 中国学生学者联合会。由于海外留学生与境外学者人数的增多，世界各地都有成立地方性的CSSA，是以促进学生学者交流及活动为目的的非盈利性、非政治性、综合性组织。旨在提高学生学者的学习生活质量，推广中国传统文化，促进中英学术和文化交流，为中国学生学者提供展示才华和追求梦想的平台，并与当地工作、生活的其他华人华侨以及华人社团保持着长期紧密的合作。

2.联合会缺乏有效的线上管理。各会内部管理工作还存在失序、人员登陆登出紊乱等现象，管理工作纷繁，对于档案信息及活动状况无详细备案，没有科学的管理方式。

3.世界各地联合会由于地域原因而过于分散，会与会之间的沟通交流存在障碍，当下网络时代需要有更有效的交流方式。

二、国内外现状

（一） 针对我国境外各地方CSSA内部管理繁杂与外部交流障碍的境况，进一步加强线上沟通交流，方便统一管理。

２、组织本国学生学者成立联合会，有利于增强民族认同感。联合会要从物质上、思想上和心理上全方位地为会员提供帮助，在异国他乡的大环境的下，使得中国学生感受到来自家乡的氤氲，增强自信，建立自我意识里的积极肯定，从而增强对国家的认同感。

（一）现状：
１、CSSA是面向在境外进修的中国公派和自费的留学人员(本科生，研究生，进修人员，访问学者，高级访问学者及项目交流、合作科研人员) 的群众性组织。成员在其中自我教育，自我管理，自我服务，合法权益受到联合会的维护。留学人员通过开展各项有益的活动，丰富了的生活；增进了留学人员之间的友谊和相互了解，加强团结；留学人员奋发学习，为国服务，同时向外国朋友宣传中华民族的文化历史，促进了中外友好关系。

２、 CSSA在世界范围内具有极其庞大数量的分会存在，诸多分会间联系松散，分会管理繁杂，会员管理亦是如此。
（二）存在的问题：
1、管理不便，缺少系统的管理途径与方法，仅仅具有线下管理机制，缺乏在线动态边界管理。
2、消息传达的不及时。当活动公告内容更新不能及时发布到各会员手中时，他们往往忽视此次活动。
3、在线交流障碍。作为群众型组织，联合会会员往往不常见
面，尤其是在如此网络环境下，学生学者更加注重在线平台的交流，
而不仅仅是线下的一些活动。

三、主要研究内容

CSSA门户网站的开发主要实现：
1.友好的首页设计：
使用WebGL技术通过threeJS和openstreetmap来3D建模
类似谷歌地球的用户体验

2.个人用户的操作：
登陆注册（填写基本信息）
查询联合会（展示联合会公开信息）
加入联合会（填写详细信息）

3.联合会会员的操作：
退出联合会、包含个人用户操作、活动报名、接收会内公告
对会内活动和公告进行留言评论

4.联合会管理员操作：
管理会员、发布活动、管理活动参与人员、发布公告
管理留言评论

5.系统管理员操作
最高权限操作、数据库更新、页面更新、网站维护
权限管理、抓取已存在CSSA页面信息与链接

6.后台数据库设计
测试实验阶段采用关系型数据库（轻量级的MySQL）
用户信息库、页面内容库、查询信息摘要库、抓取信息库
操作记录库、留言评论库、回收库（都具备良好的关系模型）

四、可行性分析

  • 资源可行性

    • 人力资源：精通web前后端开发的工程师和系统分析师
    • 软件资源：服务器平台 linux;客户端平台 浏览器。
    • 时间资源：预计项目开发、研究，与产品测试所需时间为两个月

  • 经济可行性

    • 基础投资：
      • 房屋和办公设施
      • 计算机软/硬件设备
      • 安全与保密协议
    • 直接投资：
      • 开发维护工资成本
      • 建设费、租金、维护费

  • 技术可行性

    • 关键技术 多JS框架结合 后台django
    • 开源接口 辅助

  • 社会可行性

    • 法律可行性：该产品没有侵权或者抄袭等违法情况，也没有被申请过专利，故可行。
    • 政策可行性：无国家政策限制，也无地方政府（或其它机构）的限制，故可行。
    • 使用可行性：对于服务端的操作人员，由于软件设计的提供给操作人员的接口仅仅会涉及到简单的文件新建、修改、复制、删除等操作，因此仅仅需要操作人员熟悉简单的电脑操作即可，不需要专门进行培训。 该产品操作简单快捷，功能大部分齐全，可以满足用户的基本需求，而且通俗易学。该产品不影响公司工作制度，故可以使用该产品。

        六、系统设计原则

        系统设计总原则

        为确保系统的建设成功与可持续发展，在系统的建设与技术方案设计时我们遵循如下的原则：
        1、统一设计原则统筹规划和统一设计系统结构。尤其是应用系统建设结构、数据模型结构、数据存储结构以及系统扩展规划等内容，均需从全局出发、从长远的角度考虑。
        2、先进性原则系统构成必须采用成熟、具有国内先进水平，并符合国际发展趋势的技术、软件产品和设备。在设计过程中充分依照国际上的规范、标准，借鉴国内外目前成熟的主流网络和综合信息系统的体系结构，以保证系统具有较长的生命力和扩展能力。保证先进性的同时还要保证技术的稳定、安全性。
        3、高可靠/高安全性原则系统设计和数据架构设计中充分考虑系统的安全和可靠。
        4、标准化原则系统各项技术遵循国际标准、国家标准、行业和相关规范。
        5、成熟性原则系统要采用国际主流、成熟的体系架构来构建，实现跨平台的应用。
        6、适用性原则保护已有资源，急用先行，在满足应用需求的前提下，尽量降低建设成本。
        7、可扩展性原则信息系统设计要考虑到业务未来发展的需要，尽可能设计得简明，降低各功能模块耦合度，并充分考虑兼容性。系统能够支持对多种格式数据的存储。

业务应用支撑平台设计原则

业务应用支撑平台的设计遵循了以下原则：

1、遵循相关规范或标准遵循J2EE、XML、JDBC、EJB、SNMP、HTTP、TCP/IP、SSL等业界主流标准
2、采用先进和成熟的技术系统采用三层体系结构，使用XML规范作为信息交互的标准，充分吸收国际厂商的先进经验，并且采用先进、成熟的软硬件支撑平台及相关标准作为系统的基础。
3、可灵活的与其他系统集成系统采用基于工业标准的技术，方便与其他系统的集成。
4、快速开发/快速修改的原则系统提供了灵活的二次开发手段，在面向组件的应用框架上，能够在不影响系统情况下快速开发新业务、增加新功能，同时提供方便地对业务进行修改和动态加载的支持，保障应用系统应能够方便支持集中的版本控制与升级管理。
5、具有良好的可扩展性系统能够支持硬件、系统软件、应用软件多个层面的可扩展性，能够实现快速开发/重组、业务参数配置、业务功能二次开发等多个方面使得系统可以支持未来不断变化的特征。
6、平台无关性系统能够适应多种主流主机平台、数据库平台、中间件平台，具有较强的跨系统平台的能力。
7、安全性和可靠性系统能保证数据安全一致，高度可靠，应提供多种检查和处理手段，保证系统的准确性。针对主机、数据库、网络、应用等各层次制定相应的安全策略和可靠性策略保障系统的安全性和可靠性。
8、用户操作方便的原则系统提供统一的界面风格，可为每个用户群，包括客户，提供一个一致的、个性化定制的和易于使用的操作界面。
9、应支持多CPU的SMP对称多处理结构
